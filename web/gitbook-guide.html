GUIDE Publishing Your Workshop on GitBook

From Files to
Live Website

A complete guide · GitHub → GitBook → Public URL

⏱ Total time: approximately 20 minutes from scratch
1

Set Up Your GitHub Repository

GitBook syncs from GitHub. Your repo is the source of truth — every push to GitHub automatically updates your live site.

Go to github.com → New Repository and create a public repo named:

indaba-health-ts-workshop

Initialize with a README. Make sure it is Public — GitBook's free tier requires public repos.

💡 TIP: Use this exact name. It will become your URL: yourname.gitbook.io/indaba-health-ts-workshop
2

Upload Your Workshop Files

Upload the files from this workshop package to your repository. Your final folder structure should look like this:

Repository Structure
indaba-health-ts-workshop/ ├── README.md ← Home page ├── SUMMARY.md ← Navigation (GitBook reads this) ├── docs/ │ ├── 01-motivation.md │ ├── 02-core-concepts.md │ ├── 03-hands-on.md │ ├── 03a-task1.md │ ├── 03b-task2.md │ ├── 03c-task3.md │ ├── 04-frontiers.md │ └── 05-resources.md ├── notebooks/ │ └── temporal_health_workshop.ipynb └── assets/ └── images/
📌 NOTE: SUMMARY.md is the most important file. GitBook reads it to build your sidebar navigation. Every page must be listed there.
3

Create a GitBook Account

Go to gitbook.com and click Sign up. Use your GitHub account — this makes the connection seamless.

Once logged in, click New Space. Name it "Temporal Health Intelligence — Indaba Workshop". Choose Public visibility.

4

Connect GitHub to GitBook

This is the key step. Inside your GitBook Space, go to:

Space Settings → Integrations → GitHub Sync

Click Connect GitHub. Authorize GitBook to access your repositories. Select your repo:

Repository: your-username/indaba-health-ts-workshop Branch: main Sync direction: GitHub → GitBook Root path: /

Click Sync. GitBook will immediately import all your markdown files and build the site structure from your SUMMARY.md.

💡 After setup: any git push to main will automatically update the live site within ~30 seconds
5

Customize the Appearance

In your Space, go to Customization → Appearance. Recommended settings for the workshop:

Theme: Dark Primary color: #4ade80 ← matches workshop green Logo: Upload an Indaba or workshop logo Cover image: A banner image (1200×300px works well) Favicon: Small square icon

Under Page Options, enable:

  • Page ratings (let participants give feedback)
  • Table of contents visible by default
  • Previous / Next navigation at the bottom of each page
6

Embed Your Colab Notebook

In the Hands-On Coding page in GitBook's editor, place your cursor and type /embed. Select the Embed block.

Paste your Google Colab URL. GitBook renders a clickable preview button directly in the page. Participants never need to leave the workshop site to open the notebook.

You can also add a Colab badge in your markdown:

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](YOUR_COLAB_URL)
7

Publish & Share

Click Publish in the top right of your Space. Select Public. GitBook will give you a URL:

https://yourname.gitbook.io/indaba-health-ts-workshop

Optional — Custom Domain: If you have a domain, go to Space Settings → Custom Domain and add a CNAME record pointing to hosting.gitbook.io. Then set your custom domain in GitBook.

📌 Share this URL in your opening slide, Indaba Slack, and WhatsApp groups the day before so participants can pre-load it.
8

Updating Content After the Workshop

Because everything syncs from GitHub, updates are simple:

# Edit any .md file on GitHub or locally, then: git add . git commit -m "Add post-workshop Q&A notes" git push origin main # GitBook updates automatically within ~30 seconds

Participants who bookmark the URL will always see the latest version. You can add a post-workshop page with recordings, Q&A notes, and follow-up resources.

✓

Pre-Workshop Checklist

  • GitHub repo is public and has all workshop files
  • SUMMARY.md lists every page correctly
  • GitBook GitHub Sync is active and last synced successfully
  • Colab notebook opens and runs without errors
  • Colab is embedded or linked from the Hands-On page
  • Workshop site URL is public (not behind login)
  • URL has been shared with participants in advance
  • Dataset is accessible (PhysioNet credential or simulated data)
  • Page ratings are enabled so you can collect feedback

You're Live 🎉

Your workshop is now a permanent, shareable resource for the African ML community. Every future Indaba participant who finds it can learn from what you built.
